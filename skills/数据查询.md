## 数据查询技能

当用户询问订单、销售、GMV 等数据相关问题时，请按以下步骤执行：

### 执行步骤
1. 先调用 `get_schema` 了解数据结构（若已知可跳过）
2. 根据用户问题，调用 `aggregate_orders` 或 `query_orders` 查询所需数据
3. 用 1-3 句话简洁回答用户问题（核心数字 + 关键洞察）
4. 在回复末尾按照下方【输出格式】输出数据块

### 数据填充规则
- **用户问近1天**：查询 2024-12-30 当天数据作为主指标；趋势图取最近 4 个月数据
- **用户问近7天/一周**：以近7天总额为主指标，趋势图展示近4个月对比
- **用户问月度**：以指定月份为主指标，趋势图展示各月趋势
- metrics 必须包含主指标（如"近1天 GMV"），可选加订单量、均单价
- delta 为与上一周期的环比（能算出时才加）
- 金额用 ¥ 前缀 + 千分位格式，如 ¥12,345
- 如果数据库 MCP 未连接，说明原因并给出示意数据

## 输出格式

在文字回复之后，必须紧跟以下格式的数据块（直接输出标签，不要用代码块包裹，确保 JSON 严格合法）：

<dataclaw-data>
{
  "type": "metric_card",
  "icon": "📊",
  "title": "数据看板",
  "metrics": [
    { "label": "近1天 GMV", "value": "¥12,345", "delta": "+5.2%", "delta_positive": true },
    { "label": "订单量", "value": "42" },
    { "label": "均单价", "value": "¥294" }
  ],
  "chart": {
    "title": "近4个月 GMV 趋势",
    "labels": ["2024-09", "2024-10", "2024-11", "2024-12"],
    "values": [80000, 95000, 110000, 120000]
  }
}
</dataclaw-data>

字段说明：
- `metrics`：指标数组，每项含 label（名称）、value（展示值）、delta（环比，可选）、delta_positive（true=正向/绿色，false=负向/红色）
- `chart.labels`：X 轴标签数组（月份、日期等）
- `chart.values`：纯数字数组，与 labels 等长，不含单位
- 用实际查询到的数据替换示例值
