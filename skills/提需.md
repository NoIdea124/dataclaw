## 提需技能

当用户希望提交一个数据需求（如某指标缺失、需要新建数据、当前数据不满足分析需求）时，按以下步骤执行：

### 执行步骤

1. **收集必选信息** — 从用户消息中提取或追问以下 **3 个必选字段**：
   - 需求标题（一句话描述要什么数据）
   - 业务背景（为什么需要这个数据，背后的业务问题是什么）
   - 业务负责人姓名

2. **询问是否直接创建** — 一旦 3 个必选字段已收集齐全，**立即询问用户**：
   > "必要信息已收集完毕，是否直接创建需求？后续可在需求详情页补充收益说明、优先级、期望完成日期、维度和指标等信息。"

   同时列出当前已有信息供用户确认。**不要在用户回复前擅自继续追问可选字段。**

3. **用户选择**：
   - **直接创建** — 立即调用 MCP，可选字段留空
   - **继续补充** — 依次询问可选字段后再创建：
     - 期望收益
     - 优先级（高/中/低，默认中）
     - 期望完成日期（可选）
     - 需要的维度（可选，如：日期、商品类目、用户等级）
     - 需要的指标（可选，如：复购率、客单价）

4. **调用 MCP 创建需求**：
   调用 `create_request` 工具，将返回结果中的 `id`、`request` 对象、`detail_url` 填入输出卡片。

5. **输出需求卡片** — 创建成功后**必须**按【输出格式】输出数据块，使用 MCP 返回的真实数据填写，不得省略。

### 特殊情况

- 如果用户在第一条消息里已包含全部必选信息，直接跳到步骤 2 询问是否创建，无需逐一追问
- 如果 MCP 未连接或调用失败：告知用户服务未启动（需运行 `node mcp-requests/server.js`），仍**必须**输出示意数据卡片
- 需求创建后提示用户点击链接跳转到需求详情页完善剩余信息

## 输出格式

**⚠️ 强制要求：每次需求创建完成后，必须在回复正文末尾紧跟输出以下数据块。禁止省略、禁止跳过、禁止用代码块包裹。所有字段必须填入真实值，不得保留示例占位符。**

用 MCP `create_request` 返回的真实数据填充以下字段：
- `id` 和 `request.id`：填 MCP 返回的需求 ID（如 `req_mn2ypflq_3mrq`）
- `request.title`：填用户的需求标题
- `request.background`：填业务背景
- `request.benefit`：填期望收益（无则留空字符串）
- `request.owner`：填负责人姓名
- `request.priority` / `priority_label`：high=高优先级，medium=中优先级，low=低优先级
- `request.due_date`：填期望日期（无则留空字符串）
- `request.dim_count` / `metric_count`：填维度/指标数量（无则填 0）
- `request.created_at`：填 MCP 返回的创建时间
- `detail_url`：填 `http://localhost:8080/metric-dict/index.html#request:<真实ID>`
