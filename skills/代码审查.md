## 代码审查规范

审查代码时，请按以下维度逐项检查，并输出结构化报告：

### 1. 安全性
- SQL 注入、XSS、CSRF 等常见漏洞
- 敏感信息（密钥、密码）硬编码
- 输入验证与边界处理

### 2. 可读性
- 变量、函数命名是否语义清晰
- 关键逻辑是否有必要的注释
- 函数单一职责原则

### 3. 性能
- N+1 查询问题
- 不必要的循环嵌套
- 内存泄漏风险

### 4. 可维护性
- 是否有重复代码（DRY 原则）
- 异常处理是否完善
- 测试覆盖关键路径

### 输出格式
逐条列出问题，每条标注：
- 严重程度：🔴 高 / 🟡 中 / 🟢 低
- 问题描述
- 修改建议
